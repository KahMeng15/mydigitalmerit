Migrate Keywords to Role Documents

⚠️ What This Migration Does

This migration moves the keywords field from the levelMetadata document to the individual role documents in roleMetadata subcollections.

Current Location (Incorrect):

  • meritValues/levelMetadata → roles.committee.{roleId}.keywords
  • meritValues/levelMetadata → roles.nonCommittee.{roleId}.keywords
  • meritValues/levelMetadata → roles.competition.{roleId}.keywords

Target Location (Correct):

  • meritValues/roleMetadata/committee/{roleId} → keywords
  • meritValues/roleMetadata/nonCommittee/{roleId} → keywords
  • meritValues/roleMetadata/competition/{roleId} → keywords

ℹ️ Migration Process

  1. Read keywords from meritValues/levelMetadata document
  2. For each role category (committee, nonCommittee, competition):
    • Extract keywords for each role
    • Update the corresponding role document in roleMetadata
    • Add keywords field to the role document
  3. Verify all keywords were migrated correctly
  4. Clean up (optional): Remove keywords from levelMetadata

Note: This migration is non-destructive. Keywords will be copied, not moved, until you confirm the cleanup step.
